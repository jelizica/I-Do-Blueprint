### 2. Wedding Planning App Design Best Practices

#### Color Choices for Wedding Apps

**From UI/UX Research** ([Figma Color Library](https://www.figma.com/resource-library/color-combinations/)):
- **Light pinks and soft greens** evoke romance, making them *"perfect for websites or apps in the wedding or event planning space."*
- **Mint blue with pink** in split complementary palettes promotes *"a romantic and whimsical feel‚Äîgreat for wedding or party planning apps."*

**Case Study: DreamWed Wedding Planner App** ([Medium](https://jenniferhandali.medium.com/peeking-behind-the-design-of-dreamwed-a-wedding-planner-app-31bb365ef313)):
- Used *"tender pastel colors and hues as well as applying gradients"*
- Set *"the appropriate mood for the general wedding theme, usually associated with delicacy, softness, and sophistication."*

**Case Study: Tubik Studio Wedding Planner** ([Tubik Studio](https://blog.tubikstudio.com/case-study-wedding-planner-ui-design-concept/)):
- Used **Salmon Pink (#FF9896)** as brand primary color

#### Productivity App Color Psychology

**Blue for Productivity** ([Hakuna Matata Tech](https://www.hakunamatatatech.com/our-resources/blog/color-psychology-in-ui-design)):
- *"Blue has been shown to have a calming effect and can improve focus and productivity."*

**Green for Calm Focus** ([Cieden](https://cieden.com/book/sub-atomic/color/choosing-colors-for-app-purpose)):

- *"A project management tool might use warmer tones in the morning to encourage focus and shift to cooler, more calming colors in the evening to reduce eye strain."*

### 4. WCAG Accessibility Standards

#### Minimum Contrast Ratios ([Chroma Creator](https://chromacreator.com/blog/wcag-accessibility-complete-guide))

- **WCAG AA (Required):**
  - Normal text: 4.5:1
  - Large text: 3:1

- **WCAG AAA (Enhanced):**
  - Normal text: 7:1
  - Large text: 4.5:1

#### Accessibility Best Practices ([UXPin](https://www.uxpin.com/studio/blog/color-schemes-for-apps/))

1. **Don't rely solely on color** to convey information
2. **Use contrast checkers** like WebAIM or Adobe Color
3. **Test with color blindness simulators** (8 types)
4. **Establish clear visual hierarchy** beyond color

**Critical Finding:** Your current Guest.confirmed (green) and Guest.declined (red) creates accessibility issues for 8% of males with deuteranopia.

### 5. SwiftUI Theming System Capabilities

#### Implementation Approaches ([Teabyte](https://alexanderweiss.dev/blog/2025-01-19-effortless-swiftui-theming))

**Option 1: Custom ShapeStyle** (Recommended)
- Centralized theme management
- Automatic environment-aware color selection
- Dot notation for easy use

**Option 2: Environment-Based Theme Manager**
- ObservableObject class tracks color schemes
- Works with UserDefaults for persistence
- Supports light/dark mode + custom themes

**Option 3: Color Asset Catalogs**
- Define colors in Xcode Asset Catalog
- Automatic light/dark variants
- Lightweight, built-in support

#### User Customization Pattern ([Better Programming](https://betterprogramming.pub/swiftui-app-theme-switch-241a79574b87))

```swift
// Typical implementation
enum AppTheme: String, CaseIterable {
    case blushRomance
    case sageSerenity
    case lavenderDream
    case terracottaWarm
    case system
}

@AppStorage("selectedTheme") var selectedTheme: AppTheme = .blushRomance
```

**Key Features:**
- User selects theme in Settings
- Persists via UserDefaults/AppStorage
- Applies app-wide via Environment
- Supports unlimited custom themes

struct ThemeSettingsView: View {

struct ThemePreviewRow: View {
